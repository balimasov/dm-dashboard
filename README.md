# DM Dashboard

Дашборд для Данжеон Майстра D&D: компактні картки персонажів партії з бойовим станом, статами, ресурсами, spell slots та нотатками.

## Запуск

```bash
npm install
npm run dev
```

Відкрийте [http://localhost:3000](http://localhost:3000).

## Сторінки

- `/` — дашборд з картками персонажів партії.
- `/settings` — додавання лінків на персонажів D&D Beyond (наприклад, `https://www.dndbeyond.com/characters/27964361`).
- `/characters/[id]/edit` — ручне редагування даних персонажа (базова інформація, бойовий стан, стати, ресурси, spell slots, нотатки).

D&D Beyond не має офіційного публічного API — при додаванні лінка застосунок намагається підтягнути дані автоматично (`/api/ddb/[id]`), а якщо не вийде (персонаж приватний, недоступний), дані заповнюються вручну на сторінці редагування.

На сторінці налаштувань персонажів можна перетягувати (drag-n-drop), щоб змінити порядок їх виведення на головному дашборді.

## Дані

Персонажі зберігаються у файловій SQLite-базі `data/dm-dashboard.sqlite` (створюється автоматично при першому запуску, не в git). При першому запуску база заповнюється демо-персонажами.

За замовчуванням база лежить у `<робоча директорія>/data`. Якщо в продакшені (напр. Railway) persistent volume змонтовано в інше місце — вкажи його шлях у змінній середовища `DATA_DIR`, інакше кожен редеплой стиратиме дані назад до демо-персонажів.

## Авторизація

Застосунок захищений одним спільним паролем (без окремих акаунтів — ним користується лише один ДМ). Встанови змінну середовища `DASHBOARD_PASSWORD`, щоб увімкнути логін-сторінку (`/login`); без неї застосунок відкритий без пароля (зручно для локальної розробки, але **обов'язково встанови її перед тим, як деплоїти застосунок кудись, доступне з інтернету**).

**Важливо для хостингу з health-check'ом (напр. Railway):** авторизація захищає всі шляхи, включно з `/`, тож health-check, який очікує `200` на головній сторінці, після ввімкнення пароля отримає `307`-редирект на `/login` і може вважати деплой нездоровим. Вкажи `/api/health` як Healthcheck Path у налаштуваннях хостингу — цей ендпоінт завжди відкритий і повертає `200` без авторизації.

Див. `.env.example` для повного списку змінних середовища.
